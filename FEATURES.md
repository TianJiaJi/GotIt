# 📸 截图答题工具 - 完整功能文档

## 🎯 程序概述

**截图答题工具**是一款智能化的OCR识别+AI答题工具，专为快速答题场景设计。

### 核心特性
- 🔥 **区域截图**：支持固定区域快速截图，提高答题效率
- 🔑 **快捷键操作**：全局快捷键支持，无需切换窗口
- 📝 **高精度OCR**：基于RapidOCR的高精度文字识别
- 🤖 **AI智能答题**：集成大语言模型直接给出答案
- 🔔 **系统通知**：Windows系统通知实时显示答案
- 🎨 **现代化界面**：扁平化设计，操作简单直观

---

## 📦 核心功能模块

### 1. 🔍 OCR识别模块 (`OCRManager`)

**功能**：高精度文字识别

**核心方法**：
- `recognize_text(image_path)`：识别图片中的文字
- `save_result(text, image_path)`：保存识别结果到文件
- `init_ocr()`：延迟初始化OCR引擎

**支持特性**：
- 多语言识别（auto/zh/en/zh-en）
- 置信度阈值过滤
- 自动保存识别结果到`outputs/ocr_results/`

**输出示例**：
```
识别到 3 个文本行:
1. "1+1等于几？"
2. "A. 1  B. 2  C. 3"
3. "请选择正确答案"
```

---

### 2. 🤖 AI答题模块 (`AIAnswerManager`)

**功能**：调用大语言模型进行智能答题

**核心方法**：
- `get_answer(ocr_text)`：根据OCR文本获取答案
- `init_ai()`：延迟初始化AI引擎

**支持模型**：
- DeepSeek：`deepseek/deepseek-chat`（推荐，性价比高）
- OpenAI：`gpt-3.5-turbo`, `gpt-4`, `gpt-4o`
- Anthropic Claude：`claude-3-haiku`, `claude-3-sonnet`, `claude-3-opus`
- 其他LiteLLM支持的模型

**答题特性**：
- 极速响应：专为答题场景优化
- 极简输出：直接给出答案，无冗余信息
- 智能格式：支持选择题、判断题、问答题等
- 错误处理：自动识别无效题目

**输出格式**：
```json
{
  "status": "success",
  "answer": "B. 2"
}
```

---

### 3. 📷 截图模块 (`ScreenshotManager`)

**功能**：屏幕截图和图片管理

**核心方法**：
- `take_screenshot(bbox=None)`：截取屏幕或指定区域
- `save_screenshot(screenshot, filename)`：保存截图到文件

**支持模式**：
- **全屏截图**：截取整个屏幕
- **区域截图**：截取指定矩形区域
- **自动保存**：保存到`outputs/screenshots/`

**文件命名**：
```
screenshot_20241204_153045.png
```

---

### 4. 🗺️ 区域管理模块 (`RegionManager`)

**功能**：管理和设置截图区域

**核心方法**：
- `set_point_by_shortcut(point_number)`：通过快捷键设置区域点
- `record_position_by_click(x, y)`：通过鼠标点击设置区域点
- `start_set_point1()` / `start_set_point2()`：进入设置模式
- `get_bbox()`：获取标准化的区域坐标
- `is_region_complete()`：检查区域是否完整设置
- `clear_region()`：清除区域设置

**设置方式**：
1. **快捷键设置**：ALT+CTRL+1/2 直接获取当前鼠标位置
2. **点击设置**：进入设置模式后点击鼠标左键

**区域格式**：
```python
# 输入：左上角(100, 100)，右下角(500, 400)
# 输出：(100, 100, 500, 400) - 标准化bbox
```

---

### 5. ⌨️ 快捷键管理模块 (`HotkeyManager`)

**功能**：全局快捷键监听和处理

**核心方法**：
- `set_hotkey_config(config)`：设置快捷键配置
- `set_hotkey_callback(callback)`：设置截图快捷键回调
- `set_region_callback(callback)`：设置区域设置快捷键回调
- `start_listener()` / `stop_listener()`：控制监听器
- `restart_listener()`：重启监听器（配置更改后）

**支持按键**：
- 修饰键：ctrl, alt, shift及其组合
- 主按键：字母键和数字键
- 特殊键：ESC（取消设置模式）

**监听机制**：
- 全局监听：无需切换到程序窗口
- 实时响应：按键事件立即处理
- 状态跟踪：实时跟踪按键状态

### 6. 🔔 系统通知模块 (`WindowsNotifier`)

**功能**：Windows系统通知显示

**核心方法**：
- `show_answer_notification(answer, status)`：显示AI答题结果
- `show_ocr_notification(ocr_text)`：显示OCR识别结果
- `test_notification()`：测试通知功能
- `is_available()`：检查通知功能是否可用

**支持系统**：
- Windows 10（10.0.10240+）
- Windows 11（10.0.22000+）

**通知特性**：
- 异步显示：不阻塞主程序
- 自动清理：通知自动消失
- 音效支持：可配置通知声音
- 应用图标：显示程序图标

### 7. ⚙️ 配置管理模块 (`ConfigManager`)

**功能**：统一管理程序配置

**配置项**：

#### 应用配置 (`app`)
- `name`：应用名称
- `version`：版本号

#### 快捷键配置 (`hotkey`)
- `modifiers`：修饰键组合（alt+shift等）
- `key`：主按键（字母或数字）

#### AI配置 (`ai`)
- `model`：模型名称
- `api_key`：API密钥
- `api_base`：API端点
- `temperature`：温度参数（0.1-1.0）
- `max_tokens`：最大输出长度

#### OCR配置 (`ocr`)
- `language`：语言设置（auto/zh/en/zh-en）
- `confidence_threshold`：置信度阈值（0.1-1.0）

#### 显示配置 (`display`)
- `show_result_popup`：是否显示结果弹窗（true/false）
  - true：显示详细结果对话框（包含OCR识别结果和AI答案）
  - false：仅通过系统通知显示答案，不弹出详细结果

**配置文件**：
- `config/config.json`：本地配置（包含敏感信息，不提交）
- `config/config.example.json`：配置示例（可提交）

## 🔄 完整工作流程

### 方式1：快捷键操作（推荐）

1. **设置答题区域**（首次使用）
   - 按 `ALT + CTRL + 1` → 设置区域左上角（当前鼠标位置）
   - 按 `ALT + CTRL + 2` → 设置区域右下角（当前鼠标位置）

2. **截图答题**
   - 定位到题目区域
   - 按 `ALT + SHIFT + Q` → 自动截图答题

3. **查看结果**
   - OCR识别完成 → AI自动答题
   - 系统通知弹出显示答案
   - 可选择打开详细结果对话框

### 方式2：界面操作

1. **主界面操作**
   - 点击"🗺️ 区域"按钮 → 设置答题区域
   - 点击"📷 截图答题"按钮 → 完成答题

2. **区域设置对话框**
   - 点击"设置点1"/"设置点2"按钮
   - 移动鼠标到目标位置
   - 点击鼠标左键确认点位
   - 点击"确定"保存设置

3. **查看结果**
   - 自动弹出OCR结果对话框
   - 显示AI答案和识别文本
   - 可复制文本或答案

## 🔧 配置管理

### 快速配置

#### 1. 创建配置文件
```bash
cp config/config.example.json config/config.json
```

#### 2. 编辑配置文件
```json
{
  "app": {
    "name": "截图答题工具",
    "version": "1.0.0"
  },
  "hotkey": {
    "modifiers": "alt+shift",
    "key": "q"
  },
  "ai": {
    "model": "deepseek/deepseek-chat",
    "api_key": "your-api-key-here",
    "api_base": "https://api.deepseek.com",
    "temperature": 0.3,
    "max_tokens": 200
  },
  "ocr": {
    "language": "auto",
    "confidence_threshold": 0.5
  }
}
```

#### 3. 启动程序
```bash
python gotit.py
```

### 配置项说明

#### 🔑 AI配置（必填）
- `model`：AI模型名称
  - 推荐使用：`deepseek/deepseek-chat`（性价比高）
  - 可选：`gpt-3.5-turbo`, `gpt-4`, `claude-3-haiku`等
- `api_key`：API密钥（必需）
- `api_base`：API端点地址

#### ⌨️ 快捷键配置（可选）
- `modifiers`：修饰键组合
  - 可选值：`ctrl`, `alt`, `shift`, `ctrl+alt`, `ctrl+shift`, `alt+shift`
- `key`：主按键
  - 可选值：字母键(a-z)、数字键(1-5)

#### 📊 OCR配置（可选）
- `language`：识别语言
  - 可选值：`auto`(自动), `zh`(中文), `en`(英文), `zh-en`(中英混合)
- `confidence_threshold`：置信度阈值
  - 范围：0.1-1.0，越高越严格

## 💡 使用技巧

### 提高效率的技巧

1. **预设答题区域**
   - 预先设置好答题区域，每次截图相同位置
   - 大幅提高答题速度和准确率

2. **自定义快捷键**
   - 设置符合个人习惯的快捷键
   - 避免与其他程序冲突

3. **批量答题**
   - 设置固定区域后，连续使用快捷键快速答题
   - 系统通知实时显示答案，无需查看结果

4. **结果管理**
   - OCR结果自动保存到`outputs/ocr_results/`
   - 截图文件保存到`outputs/screenshots/`
   - 可追溯历史答题记录

### 故障排除

#### 问题：OCR识别不准确
- **解决方案**：
  - 调整`confidence_threshold`参数
  - 确保截图清晰，文字大小适中
  - 尝试不同的`language`设置

#### 问题：AI答题速度慢
- **解决方案**：
  - 更换更快的模型（如deepseek/deepseek-chat）
  - 降低`max_tokens`参数
  - 检查网络连接

#### 问题：快捷键无响应
- **解决方案**：
  - 检查快捷键是否与其他程序冲突
  - 重启程序重新初始化监听器
  - 检查快捷键配置是否正确

#### 问题：系统通知不显示
- **解决方案**：
  - 确认Windows版本（10.0.10240+或11.0.22000+）
  - 检查通知权限是否开启
  - 安装`pip install Windows-Toasts`
